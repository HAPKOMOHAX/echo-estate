# Echo Estate

Тестовое Fullstack-приложение для поиска объектов недвижимости.

Проект реализован на Laravel Jetstream + Vue/Inertia. Приложение генерирует тестовую базу недвижимости, позволяет искать объекты через фильтры без перезагрузки страницы, поддерживает пагинацию, сортировку, сохранение состояния фильтров в URL и preview-изображения через Spatie Media Library.

## Стек

* Laravel
* Laravel Jetstream
* Inertia.js
* Vue 3
* Element Plus
* Tailwind CSS
* SQLite
* Spatie Laravel Media Library

## Возможности

* авторизация и регистрация через Laravel Jetstream;
* генерация около 10 000 объектов недвижимости через Seeder/Factory;
* поиск объектов без перезагрузки страницы;
* live-фильтрация с debounce;
* фильтрация по названию, наличию фото, количеству комнат и площади;
* пагинация результатов;
* сортировка по площади, количеству комнат, этажу и дате добавления;
* при поиске по названию дефолтная сортировка учитывает порядок: `запрос`, `запрос*`, `*запрос*`;
* сохранение состояния фильтров через GET-параметры;
* генерация preview-изображений через Spatie Media Library;
* адаптивная верстка для мобильных и desktop-экранов.

## Требования

Перед установкой должны быть доступны:

* PHP 8.2+
* Composer
* Node.js и npm
* PHP-расширение GD
* PHP-расширения для SQLite: `pdo_sqlite` и `sqlite3`

Проверить расширения можно командой:

```bash
php -m
```

## Быстрая установка через setup.sh

```bash
git clone <repository-url>
cd echo-estate

chmod +x setup.sh
./setup.sh
```

Для Windows можно использовать Git Bash.

Скрипт установит зависимости, создаст `.env`, подготовит SQLite-базу, выполнит миграции, заполнит базу тестовыми данными, создаст storage link и соберет frontend assets.

После установки запустите приложение:

```bash
php artisan serve
```

Приложение будет доступно по адресу:

```text
http://localhost:8000
```

## Ручная установка

### 1. Клонировать репозиторий

```bash
git clone <repository-url>
cd echo-estate
```

### 2. Установить зависимости

```bash
composer install
npm install
```

### 3. Создать .env

```bash
cp .env.example .env
```

Для Windows можно скопировать `.env.example` вручную и переименовать копию в `.env`.

### 4. Подготовить SQLite-базу

```bash
mkdir -p database
touch database/database.sqlite
```

В `.env` должны быть указаны SQLite и публичный диск:

```env
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
FILESYSTEM_DISK=public
APP_URL=http://localhost:8000
```

### 5. Сгенерировать ключ приложения

```bash
php artisan key:generate
```

### 6. Очистить кеш конфигурации

```bash
php artisan config:clear
```

### 7. Запустить миграции и сидеры

```bash
php artisan migrate:fresh --seed
```

Seeder создаст около 10 000 объектов недвижимости и подготовит изображения для объектов с фото.

### 8. Создать storage link

```bash
php artisan storage:link
```

### 9. Собрать frontend

```bash
npm run build
```

### 10. Запустить приложение

```bash
php artisan serve
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8000
```

## Работа с изображениями

В проекте используются локальные изображения для тестовых данных.

При сидировании Spatie Media Library прикрепляет изображения к тестовым объектам и создает preview-конверсию `thumb`. В списке объектов используется preview-изображение, чтобы не загружать оригиналы в карточках.

## Фильтрация и сортировка

Фильтры работают без полной перезагрузки страницы. При изменении фильтра Vue отправляет AJAX-запрос на `/properties/search`.

Поддерживаются фильтры:

* название;
* наличие фото;
* количество комнат;
* площадь от/до.

Состояние фильтров сохраняется в URL, поэтому после обновления страницы выбранные фильтры восстанавливаются.

При поиске по названию дефолтная сортировка работает в порядке:

```text
запрос
запрос*
*запрос*
```

То есть сначала идут точные совпадения, затем названия, начинающиеся с запроса, затем названия, где запрос встречается внутри строки.

## Возможные проблемы

### Не отображаются изображения

Проверьте, что создан storage link:

```bash
php artisan storage:link
```

Также проверьте значение `APP_URL` в `.env`:

```env
APP_URL=http://localhost:8000
```

После изменения `.env` можно очистить кеш:

```bash
php artisan optimize:clear
```

### Ошибка при генерации preview-изображений

Проверьте, что включено PHP-расширение GD:

```bash
php -m
```

Если `gd` отсутствует, включите его в `php.ini` или установите соответствующий пакет для вашей системы.

### Ошибка подключения к SQLite

Проверьте, что файл базы существует:

```bash
ls database/database.sqlite
```

И что в `.env` указано:

```env
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

Также должны быть включены расширения `pdo_sqlite` и `sqlite3`.
