# Echo Estate

Тестовое Fullstack-приложение для поиска объектов недвижимости.

Проект реализован на Laravel Jetstream + Vue/Inertia. Основная функциональность: генерация тестовой базы недвижимости, поиск объектов через фильтры без перезагрузки страницы, пагинация, сортировка, сохранение состояния фильтра в URL и генерация preview-изображений через Spatie Media Library.

## Стек

* Laravel
* Laravel Jetstream
* Inertia.js
* Vue 3
* Element Plus
* Tailwind CSS
* MySQL
* Spatie Laravel Media Library

## Возможности

* Авторизация и регистрация через Laravel Jetstream
* Страница поиска недвижимости
* Генерация около 10 000 объектов недвижимости через Seeder/Factory
* Фильтрация без перезагрузки страницы:

  * по названию
  * по наличию фото
  * по количеству комнат
  * по площади
* Live-фильтрация с debounce
* Пагинация результатов
* Сортировка результатов:

  * по умолчанию
  * по площади
  * по количеству комнат
  * по этажу
  * сначала новые
* При поиске по названию дефолтная сортировка учитывает порядок:

  * точное совпадение
  * совпадение с начала строки
  * вхождение внутри строки
* Сохранение состояния фильтров через GET-параметры
* Генерация preview-изображений через Spatie Media Library
* Адаптивная верстка от мобильных экранов до 4K

## Требования

Перед установкой должны быть доступны:

* PHP 8.2+
* Composer
* Node.js и npm
* MySQL
* PHP-расширение GD

GD необходим для генерации preview-изображений через Spatie Media Library.

Проверить наличие GD можно командой:

```bash
php -m
```

В списке расширений должно быть:

```text
gd
```

## Установка

### 1. Клонировать репозиторий

```bash
git clone <repository-url>
cd echo-estate
```

### 2. Установить зависимости

```bash
composer install
npm install
```

### 3. Создать `.env`

```bash
cp .env.example .env
```

Для Windows можно просто скопировать `.env.example` вручную и переименовать копию в `.env`.

### 4. Создать базу данных

Создайте MySQL-базу:

```sql
CREATE DATABASE echo_estate CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

Проверьте настройки подключения в `.env`:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=echo_estate
DB_USERNAME=root
DB_PASSWORD=
```

Если у вас другой пользователь или пароль MySQL, измените `DB_USERNAME` и `DB_PASSWORD`.

### 5. Проверить `APP_URL`

Для локального запуска через `php artisan serve` значение должно быть:

```env
APP_URL=http://localhost:8000
```

Это важно для корректной генерации ссылок на изображения Spatie Media Library.

Также должен быть указан публичный диск:

```env
FILESYSTEM_DISK=public
```

### 6. Сгенерировать ключ приложения

```bash
php artisan key:generate
```

### 7. Создать storage link

```bash
php artisan storage:link
```

### 8. Запустить миграции и сидеры

```bash
php artisan migrate:fresh --seed
```

Seeder создаст около 10 000 объектов недвижимости и подготовит изображения для объектов с фото. Preview-версии изображений генерируются через Spatie Media Library.

### 9. Собрать frontend

```bash
npm run build
```

### 10. Запустить приложение

```bash
php artisan serve
```

После запуска приложение будет доступно по адресу:

```text
http://localhost:8000
```

На главной странице можно зарегистрировать нового пользователя или войти в существующий аккаунт. После авторизации пользователь попадает на страницу поиска недвижимости.

## Быстрая установка через setup.sh

В проекте есть файл `setup.sh`, который выполняет основные команды установки:

```bash
chmod +x setup.sh
./setup.sh
```

Для Windows можно использовать Git Bash.

Перед запуском `setup.sh` необходимо заранее создать MySQL-базу `echo_estate` и проверить настройки подключения в `.env.example` или `.env`.

## Основные маршруты

```text
/                  Главная страница: вход/регистрация или редирект на поиск
/dashboard          Dashboard Jetstream
/properties         Страница поиска недвижимости
/properties/search  JSON endpoint для поиска и фильтрации
```

## Работа с изображениями

В проекте используются локальные изображения для тестовых данных.

При сидировании:

1. К объектам недвижимости прикрепляются локальные изображения.
2. Spatie Media Library создает preview-конверсию `thumb`.
3. В списке объектов используется preview-изображение.
4. Оригинальная картинка хранится отдельно.

Это позволяет не загружать большие изображения в карточках списка.

## Фильтрация и сортировка

Фильтры работают без полной перезагрузки страницы. При изменении фильтра Vue отправляет AJAX-запрос на `/properties/search`.

Поддерживаются фильтры:

* название
* наличие фото
* количество комнат
* площадь от/до

Состояние фильтров сохраняется в URL, поэтому после обновления страницы выбранные фильтры восстанавливаются.

При поиске по названию дефолтная сортировка работает в порядке:

```text
запрос
запрос*
*запрос*
```

То есть сначала идут точные совпадения, потом названия, начинающиеся с запроса, затем названия, где запрос встречается внутри строки.

## Возможные проблемы

### Битые изображения

Проверьте значение `APP_URL` в `.env`.

Для локального запуска через `php artisan serve` должно быть:

```env
APP_URL=http://localhost:8000
```

После изменения `.env` можно очистить кеш:

```bash
php artisan optimize:clear
```

### Ошибка при генерации preview-изображений

Проверьте, что включено PHP-расширение GD:

```bash
php -m
```

Если `gd` отсутствует, включите его в `php.ini` или установите соответствующий пакет для вашей системы.

### Нет доступа к файлам из storage

Выполните:

```bash
php artisan storage:link
```
